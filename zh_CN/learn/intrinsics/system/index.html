访问外部系统信息#

command_argument_count#

名称#

command_argument_count(3) - [SYSTEM:COMMAND LINE] 获取命令行参数的数量

语法#

    result = command_argument_count()

     integer function command_argument_count() result(count)
     integer :: count

说明#

command_argument_count() 返回调用包含程序时在命令行上传递的参数数量。

参数#

无

返回#

  • count

    返回值的类型为默认_integer_。它是调用程序时在命令行上传递的参数数量。

示例#

示例程序：

program demo_command_argument_count
implicit none
integer :: count
   count = command_argument_count()
   print *, count
end program demo_command_argument_count

样本输出：

   # the command verb does not count
   ./test_command_argument_count
       0
   # quoted strings may count as one argument
   ./test_command_argument_count count arguments
       2
   ./test_command_argument_count 'count arguments'
       1

标准#

Fortran 2003 及以后

另见#

get_command(3)，get_command_argument(3)

get_command#

名称#

get_command(3) - [SYSTEM:COMMAND LINE] 获取整个命令行

语法#

   call get_command(command, length, status)

    subroutine get_command(command,length,status)
    character(len=*),intent(out),optional :: command
    integer,intent(out),optional :: length
    integer,intent(out),optional :: status

说明#

检索用于调用程序的整个命令行。

请注意，在命令行上键入的内容通常由 shell 处理。 shell 通常在将特殊字符和空格传递给程序之前对其进行处理。通常可以通过关闭通配符或引用命令行参数和/或更改默认字段分隔符来关闭处理，但这应该很少需要。

返回#

  • command

    应为 character 类型和默认类型。如果 command 存在，则将用于调用程序的整个命令行存储在 command 中。

  • length

    应为 integer 类型和默认类型。如果存在 length，则为其分配命令行的长度。

  • status

    应为 integer 类型和默认类型。如果 status 存在，则在命令成功时分配 0，如果 command 太短而无法存储命令行，则分配 -1，或者在出错时分配正值。

示例#

示例程序：

program demo_get_command
implicit none
integer                      :: COMMAND_LINE_LENGTH
character(len=:),allocatable :: COMMAND_LINE
   ! get command line length
   call get_command(length=COMMAND_LINE_LENGTH)
   ! allocate string big enough to hold command line
   allocate(character(len=COMMAND_LINE_LENGTH) :: COMMAND_LINE)
   ! get command line as a string
   call get_command(command=COMMAND_LINE)
   ! trim leading spaces just in case
   COMMAND_LINE=adjustl(COMMAND_LINE)
   write(*,'("OUTPUT:",a)')COMMAND_LINE
end program demo_get_command

结果：

     # note that shell expansion removes some of the whitespace
     # without quotes
     ./test_get_command  arguments    on command   line to   echo

     OUTPUT:./test_get_command arguments on command line to echo

     # using the bash shell with single quotes
     ./test_get_command  'arguments  *><`~[]!{}?"\'| '

     OUTPUT:./test_get_command arguments  *><`~[]!{}?"'|

get_command_argument#

名称#

get_command_argument(3) - [SYSTEM:COMMAND LINE] 获取命令行参数

语法#

     call get_command_argument(number, value, length, status)

     subroutine get_command_argument(number,value,length.status)
     integer,intent(in)                    :: number
     character(len=*),intent(out),optional :: value
     integer,intent(out),optional          :: length
     integer,intent(out),optional          :: status

说明#

检索调用包含程序时在命令行上传递的第 number 个参数。

没有任何关于参数是什么的具体说明，但实际上，除非参数被引用并且通用 shell 使用的 IFS 值（内部字段分隔符）被忽略，否则参数会在空格上拆分。

选项#

  • number

    应为 integer, number >= 0 类型的标量。如果 number = 0，value 设置为程序的名称（在支持此功能的系统上）。

返回#

  • value

    应为 character 类型和默认类型的标量。 get_command_argument 返回后，value 参数包含第 number 个命令行参数。如果 value 不能容纳参数，则将其截断以适合 value 的长度。如果命令行中指定的参数少于 number 个，value 将用空格填充。

  • length

    （可选）应为 integer 类型的标量。 length 参数包含第 number 个命令行参数的长度。

  • status

    （可选）应为 integer 类型的标量。如果参数检索失败，status 为正数；如果 value 包含截断的命令行参数，则 status 为 -1；否则状态为零。

示例#

示例程序：

program demo_get_command_argument
implicit none
character(len=255)           :: progname
integer                      :: stat
integer                      :: count,i, longest, argument_length
integer,allocatable          :: istat(:), ilen(:)
character(len=:),allocatable :: args(:)
  !
  ! get number of arguments
  count = command_argument_count()
  write(*,*)'The number of arguments is ',count
  !
  ! simple usage
  !
  call get_command_argument (0, progname, status=stat)
  if (stat == 0) then
     print *, "The program's name is " // trim (progname)
  endif
  !
  ! showing how to make an array to hold any argument list
  !
  ! find longest argument
  !
  longest=0
  do i=0,count
     call get_command_argument(number=i,length=argument_length)
     longest=max(longest,argument_length)
  enddo
  !
  ! allocate string array big enough to hold command line
  ! argument strings and related information
  !
  allocate(character(len=longest) :: args(0:count))
  allocate(istat(0:count))
  allocate(ilen(0:count))
  !
  ! read the arguments into the array
  !
  do i=0,count
    call get_command_argument(i, args(i),status=istat(i),length=ilen(i))
  enddo
  !
  ! show the results
  !
  write (*,'(i3.3,1x,i0.5,1x,i0.5,1x,"[",a,"]")') &
  & (i,istat(i),ilen(i),args(i)(:ilen(i)),i=0,count)
end program demo_get_command_argument

结果：

/demo_get_command_argument a    test  'of getting   arguments  ' "  leading"

 The number of arguments is            5
 The program's name is xxx
000 00000 00003 [./test_get_command_argument]
001 00000 00001 [a]
003 00000 00004 [test]
004 00000 00024 [of getting   arguments  ]
005 00000 00018 [  leading]

cpu_time#

名称#

cpu_time(3) - [SYSTEM:TIME] 以秒为单位返回 CPU 处理器时间

语法#

     call cpu_time(time)
     real,intent(out) :: time

说明#

返回一个 real 值，以秒为单位表示经过的 CPU 时间。这对于测试代码段以确定执行时间很有用。

由于不同处理器能够提供的内容的可变性，时间的确切定义并不精确。

如果没有可用的时间源，则将 TIME 设置为负值。

请注意，TIME 可能包含与系统相关的任意偏移量，并且可能不以 0.0 开头。对于 cpu_time，绝对值是没有意义的。仅应使用后续调用之间的差异，如下例所示。

单个结果不适用的处理器（例如，并行处理器）可能会选择提供时间为数组的附加版本。

返回#

  • time

    类型应为带有 intent(out) 的 real。它被分配了一个与处理器相关的近似处理器时间（以秒为单位）。如果处理器无法返回有意义的时间，则返回与处理器相关的负值。

    开始时间不精确，因为其目的是对代码段进行计时，如示例中所示。这可能包括也可能不包括系统开销时间。

示例#

示例程序：

program demo_cpu_time
implicit none
real :: start, finish
   !
   call cpu_time(start)
   ! put code to test here
   call cpu_time(finish)
   !
   ! writes processor time taken by the piece of code.
   print '("Processor Time = ",f6.3," seconds.")',finish-start
end program demo_cpu_time

结果：

   Processor Time =  0.000 seconds.

date_and_time#

名称#

date_and_time(3) - [SYSTEM:TIME] 获取当前时间

语法#

    subroutine date_and_time(date, time, zone, values)

     character(len=8),intent(out),optional :: date
     character(len=10),intent(out),optional :: time
     character(len=5),intent(out),optional :: zone
     integer,intent(out),optional :: values(8)

说明#

date_and_time(date, time, zone, values) 从实时系统时钟中获取对应的日期和时间信息。

不可用的时间和日期_character_ 参数返回空白。

参数#

  • date

    格式为 CCYYMMDD 的默认类型的字符串，长度为 8 或更大。

  • time

    格式为 HHMMSS.SSS 的默认类型的字符串，长度为 10 或更大。

  • zone

    格式为 (+-)HHMM 的默认类型的字符串，长度为 5 或更大，表示相对于协调世界时 (UTC) 的差异。

  • values

    一个由八个元素组成的 integer 数组，其中包含：

    • value（1）：年份

    • values(2)：月份

    • values(3)：一个月中的哪一天

    • values(4)：**与 UTC 的时差，以分钟为单位

    • values(5)：一天中的小时

    • values(6)：小时的分钟数

    • values(7) ：分钟的秒数

    • values(8)：秒的毫秒数

示例#

示例程序：

program demo_date_and_time
implicit none
character(len=8)     :: date
character(len=10)    :: time
character(len=5)     :: zone
integer,dimension(8) :: values

    call date_and_time(date,time,zone,values)

    ! using keyword arguments
    call date_and_time(DATE=date,TIME=time,ZONE=zone)
    print '(*(g0))','DATE="',date,'" TIME="',time,'" ZONE="',zone,'"'

    call date_and_time(VALUES=values)
    write(*,'(i5,a)') &
     & values(1),' - The year', &
     & values(2),' - The month', &
     & values(3),' - The day of the month', &
     & values(4),' - Time difference with UTC in minutes', &
     & values(5),' - The hour of the day', &
     & values(6),' - The minutes of the hour', &
     & values(7),' - The seconds of the minute', &
     & values(8),' - The milliseconds of the second'
end program demo_date_and_time

结果：

   DATE="20201222" TIME="165738.779" ZONE="-0500"
    2020 - The year
      12 - The month
      22 - The day of the month
    -300 - Time difference with UTC in minutes
      16 - The hour of the day
      57 - The minutes of the hour
      38 - The seconds of the minute
     779 - The milliseconds of the second

system_clock#

名称#

system_clock(3) - [SYSTEM:TIME] 从实时时钟返回数字数据。

语法#

subroutine system_clock(count, count_rate, count_max)

   integer,intent(out),optional  :: count
   integer,intent(out),optional  :: count_rate
    ! or !
   real,intent(out),optional     :: count_rate
   integer,intent(out),optional  :: count_max

说明#

system_clock 允许你通过基于处理器时钟的当前值返回与处理器相关的值，以系统上通常可用的最小时间增量的精度测量持续时间。对于每个时钟计数，clock 值都会递增 1，直到达到 count_max 值，然后在下一次计数时将其重置为零。因此，clock 是一个介于 0 到 count_max 范围内的模值。 count_rate 和 count_max 假定为常数（即使 CPU 速率在单个平台上可能会有所不同）。

count_rate 取决于系统，可能会因参数的类型而异。

如果没有时钟，或者查询时钟失败，count 设置为 -huge(count)，count_rate 和 count_max 设置为零。

system_clock 通常用于测量短时间间隔（例如，系统时钟可能是 24 小时制或测量自启动以来的处理器时钟节拍）。它最常用于测量或跟踪在代码块中花费的时间，而不是使用分析工具。

参数#

  • count

    （可选）应为 integer 标量。它根据处理器时钟的当前值分配一个与处理器相关的值，如果没有时钟，则为 -huge(count)。对于每个时钟计数，处理器相关的值都会增加 1，直到达到值 count_max 并在下一次计数时重置为零。如果有时钟，它位于 0 到 count_max 范围内。

  • count_rate

    （可选）应为 integer 或 real 标量。它被分配了一个与处理器相关的近似值，即每秒处理器时钟计数的数量，如果没有时钟，则为零。

  • count_max

    （可选）应为 integer 标量。它被分配了 COUNT 可以具有的最大值，如果没有时钟，则分配为零。

示例#

示例程序：

program demo_system_clock
implicit none
integer, parameter :: wp = kind(1.0d0)
integer :: count, count_rate, count_max
integer :: start, finish
real    :: time_read

   call system_clock(count, count_rate, count_max)
   write(*,*) count, count_rate, count_max

   call system_clock(start, count_rate)
   ! <<<< code to time
   call system_clock(finish)
   time_read=(finish-start)/real(count_rate,wp)
   write(*,'(a30,1x,f7.4,1x,a)') 'time * : ', time_read, ' seconds'

end program demo_system_clock

如果处理器时钟是 24 小时制时钟，在上午 11:30 以大约每秒 18.20648193 个滴答的速度记录时间。参考资料

      call system_clock (count = c, count_rate = r, count_max = m)

定义

      C = (11*3600+30*60)*18.20648193 = 753748,
      R = 18.20648193, and
      M = 24*3600*18.20648193-1 = 1573039.

执行命令行#

名称#

execute_command_line(3) - [SYSTEM:PROCESSES] 执行一个shell命令

语法#

   subroutine execute_command_line(command, wait, exitstat, cmdstat, cmdmsg)

    character(len=*),intent(in)  :: command
    logical,intent(in),optional  :: wait
    integer,intent(out),optional :: exitstat
    integer,intent(out),optional :: cmdstat
    character(len=*),intent(out),optional :: cmdmsg

说明#

command 参数被传递给 shell 并被执行。 （shell 通常在 Unix 系统上为 sh(1)，在 Windows 上为 cmd.exe。）如果存在 wait 并且值为 .false.，则执行命令如果系统支持它是异步的；否则，命令同步执行。

最后三个参数允许用户获取状态信息。同步执行后，exitstat 包含命令的整数退出代码，由 system 返回。如果执行了命令行（无论其退出状态是什么），cmdstat 将设置为零。如果发生错误，会为 cmdmsg 分配错误消息。

请注意，系统调用不必是线程安全的。如果需要，用户有责任确保不会同时调用系统。

当命令同步执行时，execute_command_line 在命令行完成执行后返回。否则，execute_command_line 无需等待即可返回。

参数#

  • command

    包含要执行的命令行的默认 character 标量。解释取决于编程环境。

  • 等待

    （可选）默认 logical 标量。如果 wait 存在值 .false.，并且处理器支持命令的异步执行，则命令异步执行；否则同步执行。

  • exitstat

    （可选）带有 intent(inout) 的默认类型的 integer。如果命令是同步执行的，它会被分配一个依赖于处理器的退出状态的值。否则，exitstat 的值不变。

  • cmdstat

    （可选）带有 intent(inout) 的默认类型的 integer。如果出现错误情况且 cmdstat 不存在，则会启动镜像执行的错误终止。

    如果处理器不支持命令行执行，则分配值 -1，如果发生错误条件，则分配与处理器相关的正值，或者如果没有错误条件发生，则分配值 -2 但** wait** 存在值 false 并且处理器不支持异步执行。否则，它被赋值为 0。

  • cmdmsg

    （可选）默认类型的 character 标量。它是一个 intent (inout) 参数。如果发生错误情况，则为其分配一个依赖于处理器的解释性消息。否则，它不变。

示例#

示例程序：

program demo_exec
implicit none
   integer :: i

   call execute_command_line("external_prog.exe", exitstat=i)
   print *, "Exit status of external_prog.exe was ", i

   call execute_command_line("reindex_files.exe", wait=.false.)
   print *, "Now reindexing files in the background"
end program demo_exec

Note#

因为这个内在函数是进行系统调用，所以它非常依赖于系统。它关于信令的行为取决于处理器。特别是，在 POSIX 兼容的系统上，SIGINT 和 SIGQUIT 信号将被忽略，而 SIGCHLD 将被阻止。因此，如果父进程终止，子进程可能不会同时终止。

get_environment_variable#

名称#

get_environment_variable(3) - [SYSTEM:ENVIRONMENT] 获取环境变量

语法#

  call get_environment_variable(name, value, length, status, trim_name)

   character(len=*),intent(in) :: name
   character(len=*),intent(out),optional :: value
   integer,intent(out),optional :: length
   integer,intent(out),optional :: status
   logical,intent(out),optional :: trim_name

说明#

获取环境变量name的value。

请注意，get_environment_variable(3) 不必是线程安全的。用户有责任确保环境不会同时更新。

选项#

  • name

    要查询的环境变量的名称。

    应为 character 类型和默认类型的标量。

返回#

  • value

    被查询的环境变量的值。

    应为 character 类型和默认类型的标量。 name 的值存储在 value 中。如果 value 不足以容纳数据，则将其截断。如果 name 没有设置，value 将用空格填充。

  • length

    参数 length 包含存储环境变量 name 所需的长度，如果不存在，则为零。

    应为 integer 类型和默认类型的标量。

  • status

    status 是 -1 如果 value 存在但对于环境变量来说太短了；如果环境变量不存在，则为1，如果处理器不支持环境变量，则为2；在所有其他情况下，status 为零。

    应为 integer 类型和默认类型的标量。

  • trim_name

    如果 trim_name 存在且值为 .false.，则 name 中的尾随空格很重要；否则它们不是环境变量名称的一部分。

    应为 logical 类型和默认类型的标量。

示例#

示例程序：

program demo_getenv
implicit none
character(len=:),allocatable :: homedir
character(len=:),allocatable :: var
     var='HOME'
     homedir=get_env(var)
     write (*,'(a,"=""",a,"""")')var,homedir

contains

function get_env(NAME,DEFAULT) result(VALUE)
! a function that makes calling get_environment_variable(3) simple
implicit none
character(len=*),intent(in)          :: NAME
character(len=*),intent(in),optional :: DEFAULT
character(len=:),allocatable         :: VALUE
integer                              :: howbig
integer                              :: stat
integer                              :: length
   ! get length required to hold value
   length=0
   VALUE=''
   if(NAME.ne.'')then
      call get_environment_variable( &
      & NAME, length=howbig,status=stat,trim_name=.true.)
      select case (stat)
      case (1)
       !*!print *, NAME, " is not defined in the environment. Strange..."
       VALUE=''
      case (2)
       !*!print *, &
       !*!"This processor does not support environment variables. Boooh!"
       VALUE=''
      case default
       ! make string to hold value of sufficient size
       if(allocated(VALUE))deallocate(VALUE)
       allocate(character(len=max(howbig,1)) :: VALUE)
       ! get value
       call get_environment_variable( &
       & NAME,VALUE,status=stat,trim_name=.true.)
       if(stat.ne.0)VALUE=''
      end select
   endif
   if(VALUE.eq.''.and.present(DEFAULT))VALUE=DEFAULT
end function get_env

end program demo_getenv
