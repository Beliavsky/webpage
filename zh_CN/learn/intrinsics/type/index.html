类型和种类#

这些内置函数允许将一种类型的变量显式转换为另一种类型，或者在使用多态变量时可用于基于变量类型有条件地执行代码块。

Fortran 数据类型#

Fortran 提供了五种基本的内置数据类型：

  • 整数类型

    整数类型只能保存整数值。

  • 实数类型

    存储浮点数，例如 2.0、3.1415、-100.876 等。

  • 复数类型

    复数有两个部分，实部和虚部。两个连续的浮点存储单元存储这两个部分。

  • 逻辑类型

    只有两个逻辑值：.true. 和 .false.

  • 字符类型

    字符类型存储字符串。字符串的长度可以由 len 说明符指定。如果没有指定长度，则为 1。

这些“类型”可以是许多“种类”。通常不同的数字类型占用不同的存储大小，因此可以表示不同的范围；但不同的种类可以有其它的含义。例如， character 变量可能表示 ASCII 字符或 UTF-8 或 Unicode 字符。

你也可以从这些基本类型派生出你自己的数据类型。

隐式类型#

Fortran 允许一种称为隐式类型的功能，即，你不必在使用前声明某些变量。默认情况下，如果未声明变量，则其名称的第一个字母将确定其类型：

  1. 以 i-n （“整数”的前两个字母）开头的变量名称指定 integer 变量。

  2. 所有其它变量名默认为 real 。

但是，在大多数圈子中，声明所有变量被认为是良好的编程习惯。为了强制执行，你可以使用关闭隐式类型的语句开始变量声明部分：语句

implicit none

有关详细信息，请参阅 隐式 声明。

aimag#

名称#

aimag(3) - [TYPE:NUMERIC] 复数的虚部

语法#

    result = aimag(z)

     complex(kind=KIND),elemental :: aimag

     complex(kind=KIND),intent(in) :: z

说明#

aimag(z) 产生复数参数 z 的虚部。

参数#

  • z

    参数的类型应为_complex_。

返回值#

返回值是 real 类型，带有参数的 kind 类型参数。

示例#

示例程序：

program demo_aimag
use, intrinsic :: iso_fortran_env, only : real_kinds, &
 & real32, real64, real128
implicit none
complex(kind=real32) z4
complex(kind=real64) z8
    z4 = cmplx(1.e0, 2.e0)
    z8 = cmplx(3.e0_real64, 4.e0_real64,kind=real64)
    print *, aimag(z4), aimag(z8)
    ! an elemental function can be passed an array
    print *
    print *, [z4,z4/2.0,z4+z4,z4**3]
    print *
    print *, aimag([z4,z4/2.0,z4+z4,z4**3])
end program demo_aimag

结果：

  2.000000       4.00000000000000

 (1.000000,2.000000) (0.5000000,1.000000) (2.000000,4.000000)
 (-11.00000,-2.000000)

       2.000000       1.000000       4.000000      -2.000000

cmplx#

名称#

cmplx(3) - [TYPE:NUMERIC] 复数转换函数

语法#

result = cmplx(x, y, kind)

   complex elemental function :: cmplx
   TYPE(kind=KIND),intent(in), x
   TYPE(kind=KIND),intent(in),optional, y
   integer,intent(in),optional :: kind

说明#

要将数值变量转换为复数，请使用 cmplx(3) 函数。常量可用于使用语法定义复数变量

      z8 = (1.2345678901234567d0, 1.2345678901234567d0)

但这不适用于变量。你必须使用 cmplx(3) 函数。

cmplx(x [, y [, kind]]) 返回一个复数，其中 x 转换为 real 分量。如果 x 是 complex，则 y 不得存在。如果 y 存在，则将其转换为虚部。如果 y 不存在，则虚部设置为 0.0。

cmplx(3) 和双精度#

Fortran 90 语言将 cmplx(3) 定义为始终返回类型为 complex(kind=KIND(0.0)) 的结果。

这意味着 cmplx(d1,d2)，其中 d1 和 d2 是 doubleprecision，被视为：

      cmplx(sngl(d1), sngl(d2))

doubleprecision complex numbers 需要指定精度。

Fortran 90 有必要为 doubleprecision 参数指定此行为，因为这是 FORTRAN 77 规定的行为。

因此，Fortran 90 通过添加一个额外的参数来扩展 cmplx(3) 内置函数，该参数用于指定所需类型的复数结果。

      integer,parameter :: dp=kind(0.0d0)
      complex(kind=dp) :: z8
      !
      ! NO: result is just the precision of default _real_ values
      !     because KIND parameter is not specified
      !
      ! note this was stored with default real precision
      z8 = cmplx(1.2345678901234567d0, 1.2345678901234567d0)
      print *, 'NO, Z8=',z8,real(z8),aimag(z8)
      z8 = cmplx(1.2345678901234567e0_dp, 1.2345678901234567e0_dp)
      ! again, note components are just _real_
      print *, 'NO, Z8=',z8,real(z8),aimag(z8)
      !
      ! YES
      !
      ! kind= makes it work
      z8 = cmplx(1.2345678901234567d0, 1.2345678901234567d0,kind=dp)
      print *, 'YES, Z8=',z8,real(z8),aimag(z8)

F2018 组件语法复数实体的实部和虚部可以在 f2018 中使用类似组件的语法独立访问：

复数部分代号是

designator % RE
or
designator % IM.

代号是复数类型的。

所以指定符 %RE 指定复数值的实部，指定符 %IM 指定复数值的虚部。 complex-part-designator 的类型是 real，它的种类和形状是那些代号。

以下是复数部分代号的示例：

       impedance%re           !-- Same value as _real_(impedance)
       fft%im                 !-- Same value as AIMAG(fft)
       x%im = 0.0             !-- Sets the imaginary part of x to zero

参数#

  • x 类型可能是 integer、real 或 complex。

  • y（可选；仅当 x 不是 complex 时才允许）。可能是 integer 或 real。

  • kind （可选）一个 integer 的初始化表达式，指示结果的 kind 参数。

返回值#

返回值为 complex 类型，如果指定，则其种类等于 kind。如果未指定 kind，则无论 x 和 y 的种类如何，结果都是默认的 complex 种类。

示例#

示例程序：

program demo_aimag
implicit none
integer,parameter :: dp=kind(0.0d0)
complex          :: z4
complex(kind=dp) :: z8
   z4 = cmplx(1.23456789, 1.23456789)
   print *, 'Z4=',z4
   ! using kind=dp makes it keep DOUBLEPRECISION precision
   z8 = cmplx(1.2345678901234567d0, 1.2345678901234567d0,kind=dp)
   print *, 'Z8=',z8
   ! NOTE:
   ! The following is intuitive and works without calling cmplx(3)
   ! but does not work for variables just constants
   z8 = (1.2345678901234567d0, 1.2345678901234567d0 )
   print *, 'Z8 defined with constants=',z8
end program demo_aimag

典型结果：

    Z4= (1.23456788,1.23456788)
    Z8= (1.2345678901234567,1.2345678901234567)
    Z8 defined with constants= (1.2345678901234567,1.2345678901234567)

int#

名称#

int(3) - [TYPE:NUMERIC] 通过向零截断转换为整数类型

语法#

result = int(a, kind)

 integer(kind=KIND) elemental function int(a,kind)
 TYPE(kind=KIND),intent(in),optional :: a
 integer,optional :: kind

说明#

通过向零截断转换为整数类型。

参数#

  • a

    应为 integer、real 或 complex 或 BOZ-literal-constant 类型。

  • 种类

    一个 integer 初始化表达式，指示结果的种类参数。

    如果不存在，则返回的类型是默认整数类型。

返回值#

返回应用以下规则的 integer 变量或数组：

情况:

  1. 如果 a 是 integer 类型，则 int(a) = a

  2. 如果 a 的类型是 real 且 |a| < 1，int(a) 等于 0。如果 |a| >= 1，则int(a)等于大小不超过a且符号与a符号相同的整数。

  3. 如果 a 是 complex 类型，则规则 2 应用于 a 的 real 部分。

  4. 如果 a 是 boz-literal 常量，则将其视为指定 kind 的 integer。

    最高有效位为 1 的位序列的解释取决于处理器。

如果无法以指定的整数类型表示，则结果未定义。

示例#

示例程序：

program demo_int
use,intrinsic :: iso_fortran_env, only : int8, int16, int32, int64
implicit none
integer :: i = 42
complex :: z = (-3.7, 1.0)
real :: x=-10.5, y=10.5

   print *, int(x), int(y)

   print *, int(i)

   print *, int(z), int(z,8)
   ! elemental
   print *, int([-10.9,-10.5,-10.3,10.3,10.5,10.9])
   ! note int(3) truncates towards zero

   ! CAUTION:
   ! a number bigger than a default integer can represent
   ! produces an incorrect result and is not required to
   ! be detected by the program.
   x=real(huge(0))+1000.0
   print *, int(x),x
   ! using a larger kind
   print *, int(x,kind=int64),x

   print *, int(&
   & B"111111111111111111111111111111111111111111111111111111111111111",&
   & kind=int64)
   print *, int(O"777777777777777777777",kind=int64)
   print *, int(Z"7FFFFFFFFFFFFFFF",kind=int64)

   ! elemental
   print *
   print *,int([ &
   &  -2.7,  -2.5, -2.2, -2.0, -1.5, -1.0, -0.5, &
   &  0.0,   &
   &  +0.5,  +1.0, +1.5, +2.0, +2.2, +2.5, +2.7  ])

end program demo_int

结果：

             42

    -2147483648   2.14748467E+09

nint#

名称#

nint(3) - [TYPE:NUMERIC] 最近的整数

语法#

    elemental function nint(x [, kind=NN]) result(ANSWER)
     real(kind=??),intent(in) :: x
     integer(kind=NN) :: ANSWER

说明#

nint(x) 将其参数四舍五入为最接近的整数，并保留其符号。

用户必须确保该值是返回的 kind 范围内的有效值。如果处理器不能以指定的种类表示结果，则结果是未定义的。

如果 x 大于零，则 nint(x) 具有值 int(x+0.5) 。

如果 x 小于或等于 0，则 nint(x) 的值为 int(a-0.5)。

参数#

  • x

    参数的类型应为 real。

  • 种类

    （可选）一个常量 integer 表达式，指示结果的种类参数。否则，kind 类型参数为默认的 integer 类型。

返回值#

  • 回答

    结果是最接近 x 的整数，或者如果在 x 附近有两个相等的整数，则结果是使得 integer 具有更大的量级的数。

    如果无法以指定的整数类型表示，则结果未定义。

示例#

示例程序：

program demo_nint
implicit none
integer,parameter :: dp=kind(0.0d0)
real              :: x4 = 1.234E0
real(kind=dp)     :: x8 = 4.721_dp

! basic use
   print *, nint(x4), nint(x8),nint(-x8)
   ! elemental
   print *,nint([ &
   &  -2.7,  -2.5, -2.2, -2.0, -1.5, -1.0, -0.5, &
   &  0.0,   &
   &  +0.5,  +1.0, +1.5, +2.0, +2.2, +2.5, +2.7  ])

! issues
ISSUES: block
use,intrinsic :: iso_fortran_env, only : int8, int16, int32, int64
integer :: icheck
   ! make sure input is in range for the type returned
   write(*,*)'Range limits for typical KINDS:'
   write(*,'(1x,g0,1x,g0)')  &
   & int8,huge(0_int8),   &
   & int16,huge(0_int16), &
   & int32,huge(0_int32), &
   & int64,huge(0_int64)

   ! the standard does not require this to be an error ...
   x8=12345.67e15 ! too big of a number
   icheck=selected_int_kind(ceiling(log10(x8)))
   write(*,*)'Any KIND big enough? ICHECK=',icheck
   print *, 'These are all wrong answers for ',x8
   print *, nint(x8,kind=int8)
   print *, nint(x8,kind=int16)
   print *, nint(x8,kind=int32)
   print *, nint(x8,kind=int64)
endblock ISSUES

end program demo_nint

real#

名称#

real(3) - [TYPE:NUMERIC] 转换为实型

语法#

result = real(x, kind)

说明#

real(x, kind) 将其参数 x 转换为实数类型。

参数#

  • x

    应为 integer、real 或 complex。

  • 种类

    （可选）一个 integer 初始化表达式，指示结果的种类参数。

返回值#

这些函数根据以下规则返回 real 变量或数组：

  1. 如果 x 是 integer 或 real 变量，则 real(x) 将转换为默认的 real 类型。

  2. 如果 x 是 complex 变量，则 real(x) 将转换为具有种类类型参数 x 的实数类型。

  3. 如果 x 是 complex、integer real 变量, real(x, kind) 转换为 real 类型并附带种类参数。

示例#

示例程序：

program demo_real
use,intrinsic :: iso_fortran_env, only : dp=>real64
implicit none
complex              :: zr = (1.0, 2.0)
doubleprecision      :: xd=huge(3.0d0)
complex(kind=dp) :: zd=cmplx(4.0e0_dp,5.0e0_dp,kind=dp)

   print *, real(zr), aimag(zr)
   print *, dble(zd), aimag(zd)

   write(*,*)xd,real(xd,kind=kind(0.0d0)),dble(xd)
end program demo_real

结果：

 1.00000000       2.00000000
 4.0000000000000000       5.0000000000000000
 1.7976931348623157E+308  1.7976931348623157E+308  1.7976931348623157E+308

dble#

名称#

dble(3) - [TYPE:NUMERIC] 双精度转换函数

语法#

result = dble(a)

    elemental function dble(a)
    type(real(kind=kind(0.0d0)))     :: dble
    type(TYPE(kind=KIND)),intent(in) :: a

其中 TYPE 可以是 integer、real 或 complex 和 KIND 类型支持的任何类型。

说明#

dble(a) 将 a 转换为双精度 real 类型。

参数#

  • a

    类型应为 integer、real 或 complex。

返回值#

返回值的类型为 doubleprecision。对于 complex 输入，返回值具有输入值实部的大小和符号。

示例#

示例程序：

program demo_dble
implicit none
real:: x = 2.18
integer :: i = 5
complex :: z = (2.3,1.14)
   print *, dble(x), dble(i), dble(z)
end program demo_dble

结果：

  2.1800000667572021  5.0000000000000000   2.2999999523162842

transfer#

名称#

transfer(3) - [TYPE:MOLD] 传输位模式

语法#

result = transfer(source, mold, size)

说明#

解释 source 在内存中的按位表示，就好像它是与 mold 具有相同类型和类型参数的变量或数组的表示。

这大致相当于 C 概念中将一种类型“转换”为另一种类型。

参数#

  • 资源

    应为标量或任何类型的数组。

  • mold

    应为标量或任何类型的数组。

  • size

    （可选）应为 integer 类型的标量。

返回值#

结果的类型与 mold 相同，位级表示为 source。如果存在 size，则结果是长度 size 的一维数组。如果 size 不存在，但 ** mold** 是一个数组（任何大小或形状），则结果是包含 source 的全部按位表示形式所需的最小长度的一维数组。如果 size 不存在，并且 mold 是标量，则结果为标量。

如果结果的按位表示比 source 的长，则结果的前导位对应于 source 的前导位，并且任何尾随位被任意填充。

当生成的位表示不对应于与 mold 相同类型的变量的有效表示时，结果是未定义的，并且不能保证对结果的后续操作产生合理的行为。例如，可以为 var 和 .not 创建 logical 变量。 var 似乎都是真的。

示例#

示例程序：

program demo_transfer
use,intrinsic :: iso_fortran_env, only : int32, real32
integer(kind=int32) :: i = 2143289344
real(kind=real32)   :: x
character(len=10)   :: string
character(len=1)    :: chars(10)
   x=transfer(i, 1.0)    ! prints "nan" on i686
   ! the bit patterns are the same
   write(*,'(b0,1x,g0)')x,x ! create a NaN
   write(*,'(b0,1x,g0)')i,i

   ! a string to an array of characters
   string='abcdefghij'
   chars=transfer(string,chars)
   write(*,'(*("[",a,"]":,1x))')string
   write(*,'(*("[",a,"]":,1x))')chars
end program demo_transfer

结果：

   1111111110000000000000000000000 NaN
   1111111110000000000000000000000 2143289344
   [abcdefghij]
   [a] [b] [c] [d] [e] [f] [g] [h] [i] [j]

评论#

Joe Krahn：Fortran 使用 molding 而不是 casting。

与 C 语言一样，铸造是一种就地重新解释。铸件是一种围绕物体构建以改变其形状的装置。

Fortran TRANSFER 重新解释异地数据。可以认为是成型而不是铸造。 mold 是一种将形状赋予放入其中的对象的设备。

成型的优点是数据在保存它的变量的上下文中总是有效的。在许多情况下，一个体面的编译器应该将 TRANSFER 优化为一个简单的赋值。

这种方法有缺点。定义数据类型的联合体(union)是有问题的，因为您必须知道最大的数据对象，该对象可能因编译器或编译选项而异。在许多情况下，EQUIVALENCE要有效得多，但是Fortran标准委员会似乎忽略了EQUIVALENCE在少量使用时的好处。

out_of_range#

名称#

out_of_range(3) - [TYPE:NUMERIC] 值是否不能安全转换。

语法#

  result = OUT_OF_RANGE (X, MOLD [, ROUND])

   logical,elemental               :: out_of_range
   type(TYPE,kind=KIND),intent(in) :: x
   type(TYPE,kind=KIND),intent(in) :: mold
   logical,intent(in),optional     :: round

   where TYPE may be _real_ or _integer_ of any available KIND.

说明#

out_of_range(3) 确定值 x 是否可以安全地转换为 real 或 integer 变量，其类型和种类与 mold相同。

参数#

  • x

    integer 或 real 类型的标量，用于测试它是否可以存储在 mold 类型和种类的变量中

  • mold

    应为 integer 或 real 标量。如果是变量，则不需要定义，只查询类型和种类。

  • round

    标记是否在将 xx 的值验证为和mold一样的整数值之前对其进行舍入。

    仅当 x 的类型为 real 且 mold 的类型为 integer 时，round 才能存在。

返回值#

标准:

情况 (i)：如果 MOLD 是整数类型，并且 ROUND 不存在或存在值 false，则结果为真当且仅当 X 的值是 IEEE 无穷大或 NaN，或者如果具有最大值的整数介于 0 和 X 之间（含）不能由具有 MOLD 类型和种类的对象表示。

情况 (ii)：如果 MOLD 是整数类型，并且 ROUND 存在且值为真，则结果为真当且仅当 X 的值是 nity 或 NaN 的 IEEE，或者最接近 X 的整数，或如果两个整数同样接近 X，则更大的整数不能由具有 MOLD 类型和种类的对象表示。

情况 (iii)：否则，当且仅当 X 的值是不为 MOLD 类型和种类的对象支持的 IEEE inity 或 NaN，或者如果 X 是有限数且结果为真将 X 的值四舍五入（如果合适，根据 IEEE 舍入模式）到 MOLD 类型的扩展模型的量级大于与 X 具有相同符号的最大有限数的量级，并且可由MOLD类型和种类的对象表示。

注意

MOLD 必须是标量，因为从中获取的唯一信息是它的类型和种类。允许数组 MOLD 要求它与 X 一致。ROUND 是标量的，因为允许数组舍入模式在许多处理器上会产生严重的性能问题。

示例#

示例程序：

program demo_out_of_range
use, intrinsic :: iso_fortran_env, only : int8, int16, int32, int64
use, intrinsic :: iso_fortran_env, only : real32, real64, real128
implicit none
integer            :: i
integer(kind=int8) :: i8, j8

    ! compilers are not required to produce an error on out of range.
    ! here storing the default integers into 1-byte integers
    ! incorrectly can have unexpected results
    do i=127,130
       i8=i
       j8=-i
       ! OUT_OF_RANGE(3f) can let you check if the value will fit
       write(*,*)i8,j8,' might have expected',i,-i, &
        & out_of_range( i,i8), &
        & out_of_range(-i,i8)
    enddo
    write(*,*) 'RANGE IS ',-1-huge(0_int8),'TO',huge(0_int8)
    ! the real -128.5 is truncated to -128 and is in range
    write(*,*) out_of_range (  -128.5, 0_int8)         ! false

    ! the real -128.5 is rounded to -129 and is not in range
    write(*,*) out_of_range (  -128.5, 0_int8, .true.) ! true

end program demo_out_of_range

  > -128 -128  might have expected         128        -128 T F
  > -127  127  might have expected         129        -129 T T
  > -126  126  might have expected         130        -130 T T
  > RANGE IS         -128 TO  127
  > F

selected_real_kind#

名称#

selected_real_kind(3) - [KIND] 选择实数种类

语法#

result = selected_real_kind(p, r, radix)

说明#

selected_real_kind(p, r, radix) 返回实数数据类型的 kind 值，小数精度至少为 p 位，指数范围至少为 r , 基数为 radix。

参数#

  • p

    （可选）应该是一个标量，类型为 integer。

  • r

    （可选）应该是一个标量，类型为 integer。

  • radix

    （可选）应该是一个标量，类型为 integer。

在 Fortran 2008 之前，至少应存在参数 r 或 p 之一；从 Fortran 2008 开始，如果不存在，则假定它们为零。

返回值#

selected_real_kind 返回实数数据类型的 kind 类型参数的值，其十进制精度至少为 p 位，十进制指数范围至少为 R，并具有请求的 radix。如果 radix 参数不存在，则可以返回任何基数的实数。如果有多个真实数据类型满足条件，则返回小数精度最小的数据类型。如果没有真正的数据类型与条件匹配，则结果为

  • -1 : if the processor does not support a real data type with a precision greater than or equal to p, but the r and radix requirements can be fulfilled

  • -2 : if the processor does not support a real type with an exponent range greater than or equal to r, but p and radix are fulfillable

  • -3 : if radix but not p and r requirements are fulfillable

  • -4 : if radix and either p or r requirements are fulfillable

  • -5 : if there is no real type with the given radix

示例程序：

program demo_selected_real_kind

real(kind=p6) :: x
real(kind=p10r100) :: y
real(kind=r400) :: z

   print *, precision(x), range(x)
   print *, precision(y), range(y)
   print *, precision(z), range(z)
end program demo_selected_real_kind
