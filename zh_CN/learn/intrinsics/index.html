概述#

标准文档和大多数供应商提供的内置函数描述通常非常简短，以至于内置函数的功能晦涩难懂，特别是对于不熟悉该过程的人。

通过在此处描述例程

  • 更详细

  • 有一个工作示例

  • 提供其他资源的链接（包括 fortran-lang.org 上的其他文档和 Fortran Discourse 中的相关讨论）

这些文档力求成为 Fortran 程序员的宝贵资产。

这是一个社区驱动的资源，鼓励每个人为文档做出贡献。有关贡献指南，请参阅 fortran-lang.org 上的迷你书教程 和以下版权指南。

实验性的

  • 按程序审查

  • 标题审查

  • fman(1) 一个独立的 Fortran 程序，可让您从终端界面查看文档的非图形纯 ASCII 部分。编译程序并输入“./fman –help”以获取指导。

  • 手册页 一个 gzip 压缩的 tar(1) 文件，其中包含从 markdown 文档派生的早期版本的手册页。

    以管理员身份在 Linux 平台上的典型安装（但会有所不同）：

# as the administrator
cd /usr/share
tar xvfz /tmp/fortran.tgz
cd man
mandb -c

然后该平台上的任何人都可以输入命令，例如

man sinh.3fortran     # specifically show Fortran sinh(3) documentation
man -k . -s 3fortran  # list all fortran pages
man -s 3fortran --regex '.*' |col -b # show all Fortran intrinsics

有关更多信息，请参见 man(1)（即输入“man man”）。

如果您只能在自己的 ID 上安装页面，请尝试

# as a user, placing the files in ~/man:
cd
tar xvfz /tmp/fortran.tgz
cd man
mandb -c
export MANPATH="$MANPATH:$HOME/man"
export MANWIDTH=80

仍在争论是否必须将文档限制为 ANSI 字符是否值得，以便可以生成这些格式，并且仍然存在将降价转换为正确格式的问题。
